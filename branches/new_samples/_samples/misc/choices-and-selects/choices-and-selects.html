Using <choices> in <select> and <select1>

Different ways to use <choices> in <select> and <select1>.

Ordinary select1

In the following list you cannot select items which are labels for choices. In a future release we'll make it possible to use CSS to indicate this. An author could then perhaps use indentation, or a different 'voice'.

select1 with appearance="compact"

Using appearance="compact" forces the use of a tree view.

select with appearance="compact"

appearance="compact" can also be used for multiple selections. There is a bug, however, in that the labels can be selected. We plan to make this select all children.
