Running Tests

The Selenium core has been upgraded from 0.8.0 to 0.8.2.
The reason is that we needed a fix for a problem with the waitFor* actions, where they were never actually timing out. (Does Selenium not test itself with Selenium?)

Everything should be much as it was, with the only major change being that the URLs use in driver pages are now relative to the test file, rather than relative to TestRunner.html. In the longer term this is actually better, because it makes it easier to relocate a test suite. But in the short term, until we change our URLs, it will break all of our tests. Fortunately, 0.8.2 also provides a new parameter which can be used to get around this, called baseUrl, but the problem with that is that it must be absolute. Which means that the following links will no longer work, until we modify the URLs.

Adding XForms Tests

In order to add a new test, the following procedure should be followed:

  1. Create the test form in the appropriate ././html directory.
  2. Create a document called "TestSomething.html" where Something is the name of the set of tests to run on the test form. This will contain the test instructions in tabular format:
    • instructions on writing selenese can be found here,
    • and TestOutput.html is a reasonable document to use as a kick-start/template.
  3. Add a row:
    <tr><td><a href="TestSomething.html">TestSomething</a></td><tr/>
    to the table with @id="suiteTable" in the appropriate TestSuite.html
  4. The test will now appear in the list of tests available under the Selenium TestRunner.
Important: when writing "open" tests, the path is relative to selenium/core, so in order to reach the testsuite directory, the path will begin with ../../ (Unless you map a drive.)
