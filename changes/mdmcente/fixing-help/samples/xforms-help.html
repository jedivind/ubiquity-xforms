Using XForms help

The XForms help element is used by form authors to provide some extra text that will be used to help people complete their forms. The text is displayed when the user clicks on a control, and disappears when they click on the OK button.

For each of the examples we have below, there is both the markup and a live version of the code; so click on the various controls to see how the markup behaves.

The basic format

A simple example is to add a help that uses inline text, like this: 

<xf:trigger>
  <xf:label>Delete all records</xf:label>
  <xf:help>This will remove all of your records!</xf:help>
</xf:trigger>
Delete all records This will remove all of your records!

This technique is useful for using helps in different ways for different controls. For example, a help on an input control shown like this: 

<xf:input ref="firstname">
  <xf:label>Forename:</xf:label>
  <xf:help>Please enter your first name</xf:help>
</xf:input>
Forename: Please enter your first name

Another simple example is to add a help that uses instance text, like this: 

<xf:input ref="surname">
  <xf:label>Surname:</xf:label>
  <xf:help ref="/data/message"></xf:help>
</xf:input>
Surname:

Adding XForms markup and Style Attribute

Finally, just as we can add HTML markup to the help, we can also add XForms markup and the style attribute to make the width and height of the help message better: 

<xf:input ref="age">
  <xf:label>Age:</xf:label>
  <xf:help>
    Please enter <xf:output value="concat(../firstname, ' ', ../surname)"></xf:output>'s age
  </xf:help>
</xf:input>
Note how the text in the help comes from the values in the two controls, above. Changing the values above will also change the text of the help.
